# Задача 01. Зафиксировать compatibility-матрицу

## Цель

Перед началом правок зафиксировать письменную матрицу соответствия: что из публичного API .NET входит в поддерживаемое Python subset-подмножество, а что сознательно исключается.

## Почему это нужно

- без матрицы легко смешать две разные стратегии: subset-совместимость и частичную импровизацию API
- матрица станет источником истины для тестов и review

## Область этой матрицы

- источник экспортов: `src/aspose/note/__init__.py`
- scope: только текущий публичный export из `aspose.note`
- правило статусов:
  - `требует выравнивания` - тип остаётся публичным, но его форма сейчас расходится с .NET-контрактом
  - `должен быть убран из публичного экспорта` - символ не должен входить в совместимый публичный subset

## Матрица по проблемным экспортам `aspose.note.__all__`

| Символ | Статус | Решение | Комментарий |
| --- | --- | --- | --- |
| `AsposeNoteError` | должен быть убран из публичного экспорта | убрать из `__all__` или сделать внутренним | Это Python-only базовое исключение, у которого нет прямого публичного аналога в Aspose.Note for .NET. |
| `NoteTag` | требует выравнивания | переработать публичный контракт | Сейчас наружу торчат lowercase storage-поля и почти отсутствует .NET-shaped factory/API surface. |
| `TextStyle` | требует выравнивания | переработать публичный контракт | Есть Python-only публичные storage-style поля и отсутствуют части официальной .NET-формы. |
| `RichText` | требует выравнивания | переработать публичный контракт | Отсутствует `IndexOf`, и часть публичной формы ближе к внутренней реализации, чем к .NET compatibility-типу. |
| `NumberList` | требует выравнивания | пересмотреть shape | Текущая публичная форма не выглядит как явное subset-отражение официального .NET-контракта. |
| `Outline` | требует выравнивания | пересмотреть shape | Текущая форма с `X`, `Y`, `Width` выглядит расходящейся с официальной public shape .NET-типа. |
| `Image` | требует выравнивания | убрать лишние публичные члены и сверить shape | Есть риск лишних публичных Python-only свойств, например `FilePath`, которые надо проверить и скрыть/убрать. |
| `Table` | требует выравнивания | переработать публичный контракт | Публичные члены `BordersVisible` и `ColumnWidths` не совпадают по форме с официальной .NET table model. |
| `Document` | требует выравнивания | пересмотреть shape и return types | Критичное расхождение: `GetPageHistory` сейчас shape-wise ближе к `list[Page]`, а не к .NET `PageHistory`. |
| `SaveOptions` | требует выравнивания | сделать compatibility-base type | Сейчас это обычный публичный конкретный класс, что не совпадает с .NET shape. |
| `PdfSaveOptions` | требует выравнивания | убрать Python-only поля и сохранить .NET-shaped surface | Сейчас публично торчат `TagIconDir`, `TagIconSize`, `TagIconGap`, которых нет в .NET API. |

## Правила для enum

1. Ограниченный набор enum-значений допустим как часть subset-функциональности.
2. Экспортируемые enum-значения должны совпадать по смыслу и именованию с официальным .NET API.
3. Unsupported values допустимы как отсутствующие, если они не входят в поддерживаемое subset-подмножество.

## Следствие для следующих задач

1. Сначала исправляются `SaveOptions` и `PdfSaveOptions`.
2. Затем `NoteTag`.
3. Затем `TextStyle`.
4. Затем `RichText`.
5. После этого выполняется повторный аудит остальных экспортируемых типов.

## Критерии приёмки

- существует явная матрица по всем проблемным публичным символам
- по каждому проблемному символу есть решение: выровнять или убрать
- матрица согласована с будущими compatibility-тестами
