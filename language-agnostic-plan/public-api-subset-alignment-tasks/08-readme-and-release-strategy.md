# Задача 08. Обновить README и стратегию релиза

## Цель

После выравнивания контракта обновить документацию и определить безопасную стратегию выката изменений.

## Что нужно сделать

1. Обновить README так, чтобы он описывал subset-совместимый публичный API, а не просто Aspose-shaped API.
2. Явно зафиксировать, что отсутствующие возможности допустимы, но экспортируемые типы соответствуют официальной Aspose.Note for .NET по публичной форме.
3. Принять решение по breaking changes:
   - либо следующий major
   - либо переходный релиз с deprecation-shim и предупреждениями
4. Описать миграцию для пользователей, если из публичного API убираются lowercase-поля или Python-only члены.

## Риски

1. Текущие пользователи могут зависеть от несовместимых публичных полей.
2. Расширение enum откроет наружу неподдерживаемые значения, которые должны стабильно падать.
3. Часть правок придётся сопровождать временными shim-слоями.

## Критерии приёмки

- README отражает новую compatibility-цель корректно
- стратегия релиза и миграции определена
- breaking changes описаны явно и заранее
