# Задача 05. Привести RichText к subset-совместимости

## Цель

Выровнять публичную форму `RichText` с официальным типом Aspose.Note for .NET, сохранив только совместимое subset-подмножество.

## Текущее расхождение

- `RichText` имеет форму, отличающуюся от официального контракта
- отсутствует семейство методов `IndexOf`
- в публичной поверхности есть решения, больше похожие на внутреннюю Python-реализацию, чем на compatibility-тип

## Что нужно сделать

1. Пересмотреть весь публичный контракт `RichText` и оставить только совместимое subset-подмножество.
2. Добавить `IndexOf` в Python-совместимой форме, если `RichText` остаётся публичным.
3. Сохранить официальные свойства, которые должны быть доступны:
   - `Text`
   - `TextRuns`
   - `ParagraphStyle`
   - `Tags`
   - `Alignment`
   - `Length`
   - `IsTitleText`
   - `IsTitleDate`
   - `IsTitleTime`
4. Проверить, не делает ли публичное наследование от composite-типа контракт заведомо несовместимым.
5. Удалить публичные члены без .NET-аналога, если они не являются неизбежной языковой адаптацией.

## Критерии приёмки

- публичная поверхность `RichText` является subset-подмножеством официального .NET API
- явно кастомные публичные члены убраны
- `IndexOf` доступен, если тип остаётся в публичном API
