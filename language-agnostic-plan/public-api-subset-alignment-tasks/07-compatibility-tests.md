# Задача 07. Добавить compatibility-тесты публичного API

## Цель

Зафиксировать исправленный публичный контракт тестами, которые проверяют форму API, а не полноту реализации.

## Какие тесты нужны

1. Тест публичной export-поверхности `aspose.note.__all__`.
2. Тест наличия enum-значений для `SaveFormat` и `NodeType`.
3. Тесты наличия свойств и factory у `NoteTag`.
4. Тест поведения инстанциации `SaveOptions`.
5. Тест whitelist-поверхности публичных членов `PdfSaveOptions`.
6. Тест whitelist-поверхности публичных свойств `TextStyle`.
7. Тест наличия членов `RichText`, включая `IndexOf`.
8. Тесты runtime-поведения для неподдерживаемых возможностей.

## Правила написания тестов

- проверять форму через `hasattr`, `dir`, importability и callability
- не завязывать тесты на внутреннюю storage-реализацию
- разделять shape-тесты и behavioral-тесты

## Критерии приёмки

- тесты стабильно выявляют расхождения публичного контракта
- пробелы реализации выражаются через исключения, а не через исчезновение членов из API
